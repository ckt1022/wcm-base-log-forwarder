wasmtime config 使用方法
Config 是 Wasmtime Engine 的全域設定物件。
它是用來在建立 Engine 前，決定：
用哪個 compiler（Cranelift / Winch）
啟用哪些 WebAssembly proposal / feature
執行時如何配置記憶體、stack、allocator
是否支援 fuel / epoch interruption / profiling / cache / debug
是否允許 GC / component model / threads / shared memory
是否使用 pooling allocator、memory guard、copy-on-write 初始化等
也就是說，Config 是 建立 Wasmtime 執行環境的總開關面板。

1.編譯策略 strategy(Strategy::Cranelift)   //執行速度快、品質好
2.cranelift_opt_level(speed)
3.cranelift_pcc(bool)                     //對生成碼做額外的安全驗證，特別是 sandbox 邊界相關。
                                        //更有機會防止 compiler lowering bug 造成安全問題。提高 generated code 的安全可信度。

如果你要的是「牆鐘時間超過就中斷」，實務上通常這樣做：
同步執行：開 epoch_interruption(true)，另外用一個 timer/thread 週期性呼叫 Engine::increment_epoch()；
Store 設 epoch deadline，超過就 trap。這是 Wasmtime 官方設計給「長時間跑太久要打斷」的主要機制。
